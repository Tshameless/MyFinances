# Python 量化版本路线图

## 当前状态

项目已经从最初的实验性原型收敛为一个可本地运行的日频回测工具，当前版本重点解决以下问题：

1. 真实 CSV 数据接入
2. 交易日对齐和数据校验
3. 复权价格和基准对比
4. 多因子评分与等权持仓
5. 固定周期调仓、买卖受限处理与交易成本
6. TOML 配置、因子权重覆盖与参数化运行
7. 净值、调仓日志、绩效摘要和运行清单输出
8. 参数扫描、批量归档和结果汇总
9. 基础回归测试

## 当前边界

当前版本适合做研究和本地验证，还没有覆盖以下生产级能力：

1. 实时行情抓取
2. 券商下单接口
3. 真实订单、股数、现金和最小交易单位账本
4. 涨跌停、停牌、T+1、退市和动态股票池等 A 股交易细节
5. 行业/风格暴露、复杂仓位约束和组合级风险归因
6. 数据库存储、增量数据更新和数据质量监控

## 推荐下一阶段

### 阶段 1：数据增强

1. 引入标准行情数据目录和样例数据集，明确字段契约、编码和缺失值规则
2. 区分停牌、涨跌停、不可买、不可卖和退市等状态
3. 支持动态股票池、指数成分变更和上市/退市生命周期
4. 接入 A 股、ETF、加密等不同市场的数据适配层

### 阶段 2：策略增强

1. 支持更多因子
2. 增加去极值、标准化、缺失值处理、行业中性和风险暴露约束
3. 支持自定义打分、权重模型和组合优化
4. 增加 IC、Rank IC、分组收益和因子稳定性分析

### 阶段 3：回测增强

1. 引入真实持仓账本，建模股数、现金、最小交易单位和剩余资金
2. 支持更细的成交模型、分层成本、税费和冲击成本
3. 增加组合风险指标、回撤区间、换手拆解和基准归因
4. 增加参数扫描的过拟合提示和样本内/样本外评估

### 阶段 4：工程化

1. 拆分 reporting 模块，将 CSV、JSON、SVG、HTML 和格式化逻辑分层
2. 增加 CLI、报告内容、异常路径和端到端样例的回归测试
3. 清理构建产物入库，补充发布和版本管理流程
4. 增加更完整的用户文档、数据准备指南和结果解释指南
