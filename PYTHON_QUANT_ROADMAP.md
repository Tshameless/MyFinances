# Python 量化改造说明

## 目标

当前项目原本有一套 TypeScript 多因子选股原型。现在项目已经改为 **Python-only**，后续数据处理、策略研究、回测和风控都以 Python 版本为唯一主线。

这次改造的目标不是一步到位做成完整交易系统，而是先把最关键的研究闭环稳定下来：

1. 读取行情数据
2. 计算基础因子
3. 生成调仓信号
4. 运行回测
5. 输出结果报告

## 当前阶段要做什么

本阶段聚焦第一版可运行系统，范围明确如下：

1. 新增 `python_quant` 模块，作为 Python 版量化入口
2. 使用标准库实现最小可用回测链路，避免一开始就依赖复杂第三方环境
3. 支持两种数据模式
   - `--demo`：自动生成演示数据，方便立刻运行
   - `--csv`：读取本地 CSV 行情文件，接入真实研究数据
4. 实现一个基础多因子评分模型
   - 动量因子
   - 反转/均值回归因子
   - 低波动因子
5. 实现按固定周期调仓的组合回测
6. 在 `output/python` 输出回测结果文件
7. 清理 TypeScript/Node 代码和构建产物，只保留 Python 目录

## 本次交付内容

本次会交付以下内容：

8. 输出净值结果的 CSV 文件

## 为什么先这样做

先做最小可用版本，有三个原因：

1. 先把链路跑通，后续再迭代比一开始堆复杂功能更稳
2. 标准库版本更容易直接运行，减少环境问题
3. 等策略框架稳定后，再引入 `pandas`、`numpy`、`matplotlib` 这类依赖更合理
